## 👥 角色划分与任务认领

### 🛠️ 角色 A：DevOps 与服务器运维官

**负责人：** [填写姓名]
**核心职责：打通生产环境部署链路，保障服务器稳定运行。**

* **[ ] 容器化部署 (Docker & Nginx)**
  * 编写后端 Spring Boot 的 `Dockerfile`。
  * 编写 `docker-compose.yml`，一键编排应用服务、MySQL 和 Redis。
  * 配置 Nginx，完成前端静态资源的代理与转发。
* **[ ] 服务器与安全维护**
  * 学习并使用**堡垒机**，完成服务器的内网穿透与安全登录配置。
  * 分离开发与生产配置（`application-prod.yml`），妥善管理数据库敏感密钥。
  * 监控日常服务器资源占用情况，排查并导出运行日志。

---

### 🧪 角色 B：QA 测试与性能分析师

**负责人：** [填写姓名]
**核心职责：确保系统抗压无 Bug，输出严谨的测试截图与报告。**

* **[ ] 系统压力测试 (JMeter)**
  * 搭建 JMeter 测试计划，针对高频接口（如：登录、成果查询、数据导出）进行并发压测。
  * 收集吞吐量和响应时间等关键指标。
  * **截图留档**：务必保存压测过程中的聚合报告图表，用于最终答辩展示。
* **[ ] 功能用例测试**
  * 编写核心功能测试用例表（包含操作步骤、预期结果、实际结果）。
  * 执行边界值测试和越权操作测试。
  * **截图留档**：记录各个模块正常运行的 UI 界面，以及拦截非法操作的报错提示弹窗。

---

### 📖 角色 C：代码审查与技术文档专员

**负责人：** [填写姓名]
**核心职责：把控底层代码质量，输出符合软件工程规范的交付物。**

* **[ ] 全局代码审查 (Code Review)**
  * 审查后端 Controller 和 Service 层，清理冗余代码，确保全局异常拦截器正常生效。
  * **SQL 调优排查**：检查 MyBatis 的 XML 文件，确认没有严重的 N+1 查询问题或缺失索引的全表扫描。
* **[ ] 标准化文档交付**
  * **接口文档**：核对后端代码中的 Swagger/Knife4j 注释，导出最新的离线版 API 接口文档。
  * **数据库文档**：导出最终定稿的数据表字典。
  * **架构图绘制**：使用 Visio 或 draw.io 绘制系统的核心 ER 图和关键业务流程图（如审批流转状态图）。
